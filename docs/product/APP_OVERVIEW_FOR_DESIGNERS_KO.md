# 524 모바일 앱 개요

> 앱의 모든 화면과 플로우를 설명하는 간결한 가이드. 디자이너와 비기술 팀원을 위해 작성됨.

---

## 524란?

524는 **헤어/메이크업 서비스를 원하는 고객**과 **서비스를 제공하는 아티스트**를 연결하는 뷰티 서비스 예약 앱입니다. 아티스트 찾기부터 서비스 완료 후 리뷰 작성까지 전체 과정을 처리합니다.

---

## 사용자 유형

| 사용자 | 설명 |
|--------|------|
| **고객** | 행사를 위해 헤어/메이크업 서비스를 찾는 사람 |
| **아티스트** | 헤어 및/또는 메이크업 서비스를 제공하는 전문가 |

---

## 고객 플로우

### 1. 회원가입 & 로그인

**목적**: 계정을 만들고 앱에 접속합니다.

**수집하는 정보**:
- 이름
- 이메일 & 비밀번호
- 전화번호
- 생년월일

**현재 문제점**:
- 생년월일은 수집하지만 어디에도 사용되지 않음

---

### 2. 고객 온보딩 (선택)

**목적**: 예약 전에 고객의 스타일 선호도를 파악합니다.

**수집하는 정보**:
- 닮았다고 들어본 연예인
- 원하는 스타일의 연예인
- 예쁘다고 생각하는 연예인
- 관심 있는 서비스 (헤어, 메이크업, 또는 둘 다)

**의도**: 이 정보는 아티스트가 예약 전에 고객이 원하는 룩을 이해하는 데 도움이 되어야 합니다.

**현재 문제점**:
- 이 데이터는 아티스트에게 절대 표시되지 않음
- `shareWithStylist` 설정이 존재하지만 구현되지 않음
- 결국, 질문은 하지만 답변으로 아무것도 하지 않음

---

### 3. 예약 플로우

**목적**: 서비스 선택, 아티스트 선택, 예약 확정까지 고객을 안내합니다.

#### 1단계: 위치
- 고객이 서비스 받을 장소 입력 (자택, 행사장 등)
- 위치에 대한 메모 추가 가능 (주차, 접근 방법)

**문제점**: 위치 메모는 저장되지만 누구에게도 표시되지 않음

#### 2단계: 연예인/스타일 질문 (선택 경로)
- 온보딩과 동일한 3가지 연예인 질문
- 스타일 선호도를 요약한 "결과" 화면 표시

**문제점**: 이 데이터는 예약에 저장되지 않음 - 세션 종료 후 사라짐

#### 3단계: 서비스 유형
- 고객이 선택: 헤어만, 메이크업만, 또는 둘 다

**문제점**: 이 선택은 저장되지만 예약 상세에 표시되지 않음

#### 4단계: 행사
- 서비스 목적 선택 (결혼식, 상견례, 소개팅 등)

**문제점**: 행사는 저장되지만 고객이나 아티스트의 예약 상세에 표시되지 않음

#### 5단계: 일정
- 고객이 원하는 날짜와 시간대 선택

#### 6단계: 아티스트 선택
- 필터와 정렬로 가능한 아티스트 탐색
- 평점, 경력, 전문 분야가 있는 아티스트 프로필 확인
- 아티스트 선택

#### 7단계: 시술 선택
- 선택한 아티스트의 메뉴에서 구체적인 시술 선택
- 각 시술의 가격과 소요시간 확인
- 합계 금액 표시

#### 8단계: 스타일 선택
- 고객이 원하는 룩의 참고 이미지 최대 3개 선택 가능
- 직접 사진 업로드도 가능

**문제점**: 이 이미지들은 수집되지만 서버로 전송되지 않음 - 아티스트가 볼 수 없음

#### 9단계: 결제 & 메모
- 예약 요약 검토
- 아티스트에게 특별 요청이나 메모 추가
- 확정 및 결제

**문제점**: 고객 메모는 저장되지만 아티스트에게 표시되지 않음

#### 10단계: 확인
- 예약 확정 화면과 다음 단계 안내

---

### 4. 홈 화면

**목적**: 로그인 후 고객의 메인 대시보드.

**표시되는 정보**:
- 고객 이름으로 인사
- 다음 예약까지 남은 일수
- 알림 배지

**표시되어야 하지만 안 되는 정보**:
- 예정된 예약의 아티스트 이름
- 예약된 서비스
- 총 금액
- 예약 위치

**현재 문제점**:
- "내 예약" 캐러셀이 플레이스홀더/가짜 데이터 표시
- "베스트 리뷰" 캐러셀이 플레이스홀더/가짜 데이터 표시
- "베스트 아티스트" 캐러셀이 플레이스홀더/가짜 데이터 표시

---

### 5. 예약 관리

**목적**: 예약 조회 및 추적.

**고객 예약 상세에 표시되는 정보**:
- 예약 번호 및 상태
- 날짜 및 시간
- 가격이 포함된 서비스 목록
- 총 금액
- 아티스트 이름
- 위치
- 상태 타임라인

**고객 화면에서 누락된 정보**:
- 행사 유형
- 본인이 작성한 메모/특별 요청
- 선택한 스타일 참고 이미지

---

### 6. 리뷰

**목적**: 완료된 서비스 평가 및 리뷰 작성.

**고객이 제출하는 정보**:
- 전체 평점 (1-5 별점)
- 품질 평점 (1-5 별점)
- 전문성 평점 (1-5 별점)
- 시간 준수 평점 (1-5 별점)
- 작성 리뷰 (선택)
- 사진 (선택, 최대 5장)

**리뷰가 표시되는 곳**:
- 고객은 자신이 제출한 리뷰 확인 가능
- 아티스트는 받은 리뷰와 집계 통계 확인 가능
- 아티스트는 리뷰에 답변 가능

---

### 7. 메시징

**목적**: 고객과 아티스트 간 직접 소통.

**기능**:
- 텍스트 메시지
- 이미지 공유
- 특정 예약과 연결
- 읽음 확인
- 입력 중 표시
- 전체 메시지 기록 보존

---

### 8. 알림

**목적**: 예약에 대한 정보를 고객에게 알림.

**알림 유형**:
- 예약 생성됨
- 예약 상태 변경됨
- 아티스트로부터 새 메시지

**문제점**: 알림 유형을 구분하는 시각적 아이콘 없음

---

## 아티스트 플로우

### 1. 회원가입

**목적**: 아티스트 계정 생성.

**수집하는 정보**:
- 이메일 & 비밀번호만

아티스트는 회원가입 시 최소 정보만 제공하고, 온보딩에서 프로필을 완성합니다.

---

### 2. 아티스트 온보딩 (필수)

**목적**: 예약을 받기 전에 아티스트의 공개 프로필 구축.

**1단계 - 기본 정보**:
- 활동명 (표시 이름)
- 소개/자기소개

**2단계 - 전문성**:
- 전문 분야 (헤어, 메이크업, 또는 둘 다)
- 경력 연수

**3단계 - 서비스 지역**:
- 주요 활동 지역 (지도에서 선택)
- 출장 가능 거리 (반경)

**4단계 - 프로필 사진**:
- 전문적인 사진 업로드

온보딩 완료 후, 아티스트는 관리자 승인을 기다립니다.

---

### 3. 승인 대기

**목적**: 관리자가 아티스트 프로필을 검토하는 동안의 대기 화면.

프로필이 검토 중이라는 간단한 메시지 표시.

---

### 4. 아티스트 예약 관리

**목적**: 예약 요청 조회 및 응답.

**예약 목록에 표시되는 정보**:
- 예약 번호
- 서비스 요약
- 날짜 및 시간
- 총 금액
- 상태 배지

**예약 상세에 표시되는 정보**:
- 위의 모든 정보 포함:
- 가격이 포함된 전체 서비스 내역
- 결제 상태
- 상태 히스토리 타임라인
- 액션 버튼 (수락/거절/시작/완료)
- 고객에게 메시지 버튼

**아티스트 화면에서 누락된 중요 정보**:

| 누락된 정보 | 왜 중요한가 |
|-------------|-------------|
| 고객 이름 | 누구에게 서비스하는지 모름 |
| 고객 전화번호 | 직접 연락처 없음 |
| 행사 유형 | 이벤트 맥락 없음 |
| 스타일 선호도/연예인 정보 | 원하는 룩 준비 불가 |
| 참고 이미지 | 고객이 원하는 스타일을 볼 수 없음 |
| 특별 요청/메모 | 고객 지시사항 누락 |
| 알레르기 | **안전 위험** |
| 피부 민감성 | **안전 위험** |
| 의료 메모 | **안전 위험** |

---

### 5. 아티스트 리뷰

**목적**: 고객으로부터 받은 피드백 확인.

**아티스트가 보는 정보**:
- 4개 카테고리의 평균 평점
- 총 리뷰 수
- 텍스트와 사진이 포함된 개별 리뷰
- 리뷰에 답변 기능

---

## 주요 문제점 요약

### 수집되지만 사용되지 않는 데이터

| 데이터 | 수집 위치 | 문제점 |
|--------|-----------|--------|
| 연예인/아이돌 선호도 | 온보딩 + 예약 플로우 | 아티스트에게 표시 안 됨 |
| 스타일 참고 이미지 | 예약 플로우 | 서버에 저장 안 됨 |
| 고객 메모 | 결제 | 표시 안 됨 |
| 행사 유형 | 예약 플로우 | 표시 안 됨 |
| 서비스 유형 | 예약 플로우 | 표시 안 됨 |
| 위치 메모 | 예약 플로우 | 표시 안 됨 |
| 생년월일 | 회원가입 | 목적 없음 |

### 안전 우려

앱에는 알레르기, 민감성, 의료 메모에 대한 데이터베이스 필드가 있지만:
- 고객에게 이 정보를 절대 묻지 않음
- 아티스트는 예약 전에 건강/안전 정보를 볼 방법이 없음

### 아티스트 경험

아티스트가 예약 요청을 받을 때 보는 정보:
- 서비스와 가격
- 날짜와 시간
- 위치

보지 못하는 정보:
- 고객이 누구인지
- 고객이 원하는 룩
- 특별 지시사항
- 행사가 무엇인지
- 건강/안전 정보

**이로 인해 아티스트가 제대로 준비하거나 예약 수락 여부를 결정하기 매우 어려움.**

## 디자인을 위한 권장사항

1. **아티스트 예약 상세 전면 재설계 필요** - 고객 정보, 스타일 선호도, 메모, 안전 정보 추가

2. **고객 홈 화면에 실제 데이터 필요** - 플레이스홀더 캐러셀을 실제 예약 기록과 추천으로 교체

3. **스타일/선호도 데이터 흐름 연결 필요** - 연예인 질문을 제거하거나 실제로 아티스트에게 표시

4. **건강/안전 정보 수집 추가** - 아티스트가 예약 전 알레르기를 알아야 함

5. **행사와 메모를 모든 곳에 표시** - 고객과 아티스트 모두 예약 상세에서 이 맥락을 봐야 함
